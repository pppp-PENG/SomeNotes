### 2. 线程

**线程模型**

- 主线程：接收请求 - 解析 - 数据读写 - 发送数据
- 三个后台线程：异步处理关闭文件任务、AOF 刷盘任务、释放内存任务
- 三个 I/O 线程：分担网络压力

### 3. 持久化

**AOF (Append Only File) 日志**

- 先执行请求的操作命令，再存入 AOF 缓存区中，最后由后台线程异步写入磁盘中
- 写入磁盘的方式：Always（每次操作）、Everysec（每秒）、No（操作系统决定）
- 当日志过大时，触发重写机制，压缩日志（每个键值只存储最新的命令）

**RDB (Redis Database) 快照**

- 分为 save 和 bgsave
- bgsave 的实现
    - 首先 fork() 创建子进程
    - 若主进程进行读操作，则互不影响
    - ==若主进程进行写操作，数据会在内存中复制一份副本==，bgsave 以副本为准写入磁盘

**混合持久化方式**

- 降低 RDB 快照的频率，每两次 RDB 快照之间的操作都用 AOF 日志记录

### 5. 功能

**过期删除策略**

- 惰性删除策略：不主动删除，每次访问 key 时检测是否过期，若过期则删除
- 定期删除策略：每隔一定时间随机抽取 key 检查是否过期并删除过期的 key，==若此次检查过期占比大于25%，则再抽一次==，此种循环时间上限一般不超过25ms
- AOF 删除策略：
    - ==写入 AOF 文件时不会检查过期键==，但会存储删除过期键的指令
    - AOF 文件重写时会进行过期检查，不加载过期键
- RDB 删除策略：
    - 生成 RDB 文件时会进行过期检查，不保存过期键
    - 加载 RDB 文件时主服务器会进行过期检查，不加载过期键
- 主从模式删除策略：==从库并不会主动进行过期检查==，被动接收主库的控制
- 内存淘汰：（默认不进行内存淘汰）
    - volatile-random 随机淘汰设置了过期时间的键值
    - volatile-ttl 优先淘汰设置了过期时间且最早过期的键值
    - volatile-lru 优先淘汰设置了过期时间且最久未使用的键值
    - volatile-lfu 优先淘汰设置了过期时间且最少使用的键值
    - allkeys-random 随机淘汰任意键值
    - allkeys-lru 优先淘汰最久未使用的键值
    - allkeys-lfu 优先淘汰最少使用的键值


### 6. 缓存

**缓存雪崩**

缓存雪崩：当大量数据在同一时间过期时，==大量针对过期数据的请求==会直接到达数据库

缓存击穿：当热点数据过期时，==大量针对过期热点数据的请求==会直达数据库

缓存穿透：==大量针对一个不存在的数据的请求==会直达数据库
