### 1. Java基础

**Java 三大特点**

==平台无关性、面向对象、内存管理==

注：字节码与平台无关，但机器码因 CPU 不同需要用不同的编码方式

**常用简称**

- Java 虚拟机（Java Virtual Machine）

- Java 运行环境（Java Runtime Environment）

- Java 开发工具（Java Development Kit）


### 2. 数据类型

**Java 中的包装类**

- 把数据跟处理这些数据的方法结合在一起
- 在 Java 中绝大部分方法或类都是用来处理类类型对象的

- 泛型应用——泛型只能使用类不能使用基本数据类型
- 类型转换—— `int -> String` 需要先转成 `Integer` 才能转成 `String`
- 集合应用——集合只能存对象，不能存基本数据类型

**`Integer` 与 `int`**

`int` 是基本数据类型，只存在对应内存块中，节约内存

`Integer` 创建时既会创建引用也会创建对象；==若值为 -128~127，则会复用同一对象，不创建新对象==

**`BigDecimal` 类**

十进制数字

方法：`add`、`subtract`、`multiply`、`divide (val, 保留小数位数, 舍入模式)`

### 3. 面向对象

**面向对象三大特点**

- ==封装==：对外隐藏对象的内部细节，仅通过对象提供的接口与外界交互【增加安全性，简化编程，独立对象】
- ==继承==：使子类自动共享父类数据结构和方法【代码复用，结构清晰】
- ==多态==：使用父类的方法，但不同的子类实例对象会有不同的执行内容（子类重写了父类方法）【灵活性，拓展性】

**常用修饰关键字**

`public`, `protected`, `default`, `private`; `abstract`, `static`, `final`

注：`final` 的作用有：增加代码可读性、JVM 内联优化性能

**抽象类和接口类**

- 类：都不能实例化，且抽象类不能加 `final` 关键字（因为抽象类需要被继承）
- 方法：接口只能有抽象方法或==默认方法==（接口唯一能写具体实现的方法），==且接口无构造函数==

（可以把抽象类比喻成一种类的模板，而接口类可以比喻成一种标准规定；模板可以有构造方法，而规定不会限制构造方法）

注：内部类作用是可以访问私有成员

**静态（`static`）**

- 静态变量：类加载时初始化，只分配一次内存
- 静态方法：不可重写，但可被==隐藏（父子类的同名静态方法同时存在，看引用类型决定用哪个方法）==

**Object 类**

`equals` 方法在未重写的情况下和 `==` 一样，比较对象的内存地址

`String` 类重写了 `equals` 方法，比较对象的内容是否相等

**String 相关**

`String` 被声明了 `final`，因此使用 `+` 拼接时会自动转换成 `StringBuilder` 拼接后转回来

`StringBuffer` 有字符串拼接方法，并且每个方法都被 `synchronize` 关键字修饰，线程安全

`StringBuilder` 去掉了线程安全的部分，提升了拼接效率

**Java 8**

Stream API：是用于处理数据集合（比如集合、数组等）的新抽象层，支持顺序和并行处理（使用 `Parallel()` 方法），并且可以支持复杂的数据转换操作，还可以将多个操作链接起来形成一个操作管道

Lambda 表达式：一个匿名函数，没有名称，但有参数列表、函数主体和返回类型（可以简洁地表示一个只有单个抽象方法的接口的实例，(参数) -> {方法体} 表示一个函数，可以作为 Stream 流的 .filter() 方法的参数）

**泛型**

- 代码复用：让多种数据类型执行相同的代码
- 类型安全：编译时检查类型

**反射**

反射根本意义在于能==在运行时使用编译器未知的类==！

- 创建对象：`Class` 类的 `newInstance()` 方法、`Constructor` 类的 `newInstance()` 方法
- 调用方法：`Method` 类的 `invoke()` 方法
- 访问修改属性：`Field` 类的 `get()` 和 `set()` 方法

**注解**

作用：编译检查（`@Override` 检查是否正确重写）、动态处理（Spring 中常用）

### 4. 异常

**异常分类**

1. `Error`：程序无法处理的严重问题，如系统崩溃、虚拟机错误、动态链接失败等。如 `OutOfMemoryError`、 `StackOverflowError` 等
2. `Exception`：程序本身可以处理的异常条件
    - 非运行时异常：这类异常在编译时期就必须被捕获或者声明抛出。它们通常是外部错误，如 `FileNotFoundException`、`ClassNotFoundException` 等。非运行时异常强制程序员处理这些可能出现的问题，增强了程序的健壮性
    - 运行时异常：运行时异常由程序错误导致，如 `NullPointerException`、`lndexOutOfBoundsException` 等。运行时异常是不需要在编译时强制捕获或声明的

**异常处理**

1. `try - catch - finally` 语句块
2. `throw` 语句和 `throws` 关键字

### 5. Java 虚拟机

堆用于存储创建的实例对象和数据

### 6. 线程

创建线程：写一个继承 `Thread` 类的子类、写一个实现 `Runnable` 接口的类，重写其中的 `run()` 方法

停止线程：自我停止—— `run()` 方法里的 `return` 便可以停止；他人停止——其他线程调用该线程的 `interrupt()` 方法

`Object.wait()` 方法表示本线程释放该对象的锁，并进入等待状态

`Object.notify()` 方法表示激活等待这个对象的锁的随机一个线程
