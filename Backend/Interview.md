### Java

1. **数组和 HashMap 的底层实现，数组如何扩容，HashMap 是否线程安全，ConcurrentHashMap 如何做到线程安全的（平安科技）**

底层实现：连续分配内存创建 Entry[] 数组，元素为链表头；HashMap 连续分配内存（散列桶数组），用 key 的 HashCode 值来决定到数组哪个位置

数组扩容：扩容成1.5倍，长度右移再加上原长度（原长度 += 原长度 >> 1）

线程安全：HashMap 线程不安全，若要进行并发操作，可以使用 ConcurrentHashMap（它开辟了 Node 数组，保证每个 Node 头节点的同步访问）

2. **重写和重载的区别（科荣）**

重写：在继承关系中，子类提供了一个与父类中同名且具有相同参数列表的方法，以实现不同的行为

重载：在同一个类中，可以存在多个方法名相同但参数列表不同的方法

Collection 接口（List、Set、Queue）、Map 接口

HashSet（key存数据，value值不变的HashMap）、LinkedHashSet、TreeSet（元素有序）

PriorityQueue、LinkedList

4. **如何创建一个线程，Java 中的线程占有哪些资源（科荣）**

创建 Thread 对象，调用 start() 方法时便创建了新线程（两种方案，第一种是写一个继承 Thread 的子类，重写 run() 方法；第二种是写一个实现 Runnable 接口的子类，重写 run() 方法）

线程在JVM中占有程序计数器、虚拟机栈和本地方法栈等资源，主内存中还有其相对应的线程控制块等

5. **JDK 8 的新特性（科荣）**

Lambda 表达式：一个匿名函数，没有名称，但有参数列表、函数主体和返回类型（可以简洁地表示一个只有单个抽象方法的接口的实例，(参数) -> {方法体} 表示一个函数，可以作为 Stream 流的 filter() 方法的参数）

Stream API：是用于处理数据集合（比如集合、数组等）的新抽象层，支持顺序和并行处理（使用 Parallel() 方法），并且可以支持复杂的数据转换操作，还可以将多个操作链接起来形成一个操作管道

6. **JVM是什么，其内存模型（华为 OD、平安科技）**

JVM（Java 虚拟机）是一种可以执行 Java 字节码的虚拟机进程。它是 Java 平台的核心组成部分，负责运行 Java 应用程序

其内存模型如下

- 线程私有的有：程序计数器（用来存储指向下一条指令的地址，在执行 Java 方法时，它存储的是正在执行的虚拟机字节码指令的地址；而在执行 Native 方法时，这个计数器值为空）、Java 虚拟机栈（ 每个方法在执行的同时会创建一个栈帧用于存储局部变量表、操作数栈、动态链接、方法出口等信息）、本地方法栈（为虚拟机使用到的 Native 方法服务）

- 线程共享的有：Java 虚拟机堆（存放对象实例和字符串常量池）、元空间（存储类的元数据、常量池、字段和方法数据等信息）、直接内存（不是 Java 虚拟机规范中定义的内存区域，常用于与本地代码（如 C/C++ 库）进行交互，可避免在 Java 堆和 Native 堆中来回复制数据）

7. **Java 的 I/O 模型（华为 OD）**

- 同步阻塞I/O（BIO）：基于流（Streams）和通道（Channels）的概念进行
- 同步非阻塞I/O（NIO）：可以构建多路复用的、同步非阻塞 IO 程序，提供了 Channel、Selector、Buffer 等新的抽象，轮询等待而非阻塞
- 异步非阻塞I/O（AIO）：基于事件和回调机制，操作后直接返回，等操作系统的通知

8. **Java 的线程池及应用（华为 OD、平安科技）**

线程池由`java.util.concurrent`包提供，是允许重用线程，减少创建和销毁线程的开销，以及控制并发线程的数量

9. **hashCode 方法，equals 方法和 "=="（平安科技）**

hashCode 方法返回根据内存地址计算的哈希值

equals 方法在未重写（String 类有重写）的情况下，它比较的是对象的内存地址。但在多数情况下，这个方法会被重写，以比较对象的内容或属性是否相等。

"==" 比较内存中的存放的值是否相等（引用类型变量在内存中存放的是引用对象的地址）

而 hashCode 方法返回的哈希值可能存在哈希冲突，因此不能代替 equals 方法。

### Spring

控制反转（IoC）：Spring 通过 IoC 将对象实例的创建和依赖关系的管理交由 Spring 容器负责，从而降低了组件之间的耦合度

MVC 框架：Spring MVC 为构建 Web 应用程序提供了一个分离的模型，它通过 DispatcherServlet 来处理请求，并通过视图技术（如 JSP、Thymeleaf 等）来生成响应

### 数据库

1. **数据库三大范式（平安科技）**

第一范式（1NF）：要求数据库表的每一列都是不可分割的原子数据项。【列不可分割】

第二范式（2NF）：非键属性必须完全依赖于候选键（消除非主属性对主码的部分函数依赖）【非键属性不能只被主键中的部分属性决定，这个“部分属性”可以作为主键和该非键属性形成新表】

第三范式（3NF）：任何非主属性不依赖于其他非主属性（消除传递依赖）【非键属性不能被其他非键属性决定，这个“其他非键属性”可以作为主键和该属性形成新表】

2. **为什么大厂不用外键（平安科技）**

因为外键要保持两张表的一致性，如果数据量十分大，采用了分布式的数据库存储数据，那么每次更新数据都需要检查外键约束，一旦发生大量的更新操作，容易使数据库性能急剧下降或者发生死锁。因此外键适合单机低并发的业务，而不适合分布式高并发的业务

GET 通常用于请求服务器返回资源。它应该是幂等的，意味着多次执行相同的GET请求应该返回相同的结果，不会对服务器状态产生副作用

### 操统

1. **Nginx 反向代理是什么（比亚迪）**

Nginx 反向代理允许 Nginx 服务器接收客户端的请求，然后将这些请求转发到其他服务器（通常是后端服务器），并将从后端服务器返回的响应结果再发送给客户端。在这个过程中，客户端并不知道自己的请求实际上是由后端服务器处理的，它只知道与 Nginx 服务器进行了交互

优点：

- 负载均衡：通过将请求分配到不同的后端服务器实现负载均衡，提高服务的可用性和扩展性
- 缓存：缓存后端服务器的响应内容，从而减少后端服务器的负载，加快响应速度
- 安全性：隐藏后端服务器的真实地址，提供一层额外的安全防护

- 进程（Process）：操作系统资源分配的基本单位
- 线程（Thread）：任务调度和执行的基本单位

### 数据结构

1. **二叉树和平衡二叉树的区别，如何先序遍历二叉树（比亚迪）**

- 二叉树：是每个节点最多有两个子树的树结构。通常子树被称作“左子树”（left subtree）和“右子树”（right subtree）
- 平衡二叉树（AVL树）：是一种特殊的二叉搜索树，它或者是一棵空树，或者是具有下列性质的二叉树：它的左子树和右子树都是平衡二叉树，且左子树和右子树的高度之差的绝对值不超过1
- 先序遍历二叉树的操作顺序是：先访问根节点，然后递归地遍历左子树，最后递归地遍历右子树

### Docker

Docker 是一个开源的应用容器引擎，它允许开发者打包他们的应用以及应用的依赖包到一个可移植的容器中，然后发布到任何流行的 Linux 机器上

Docker 镜像（Docker Image）是 Docker 容器运行时的只读模板，包含启动容器所需要的所有文件系统结构和内容

- Docker 镜像是构建 Docker 容器的模板。每个容器在启动时都是基于一个镜像创建的
- 镜像可以创建多个容器实例，这意味着同一个镜像可以被用来启动多个容器，每个容器都是镜像的一个运行实例

### 分布式

1. **分布式的 CAP 理论（比亚迪）**

在分布式计算环境中不可能同时保证以下三个特性：

- 一致性（Consistency）：每次读取都能获得最新的写入数据或者错误信息
- 可用性（Availability）：每个请求都能得到一个（非错误）响应，不管这个响应是成功还是失败
- 分区容错性（Partition tolerance）：系统在遇到网络分区（即部分节点之间的通信不可达）时，仍然能够继续运作
